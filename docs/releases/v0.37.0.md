# TypeScript WSDL Client v0.37.0

## Faster Release Publishing

This release keeps the full release gate before tagging and makes the post-release package workflow more focused.

## What This Improves

Publishing from an already validated release tag no longer reruns the full CI, conformance, and smoke pipeline inside the package workflow. The workflow now performs the publish-specific checks needed before npm and GitHub Packages publication: build, typecheck, agent skill validation, and package contents validation.

Release preflight now also verifies that the publish workflow stays targeted. This keeps the release safety boundary explicit: run full CI and conformance before pushing the tag, then run package-specific validation from the immutable tag before publishing.

## Highlights

- Adds `npm run release:publish-check` for targeted post-release package validation.
- Updates the package publishing workflow to use the targeted publish check.
- Adds release preflight coverage for the publish workflow gate.
- Updates the 1.0 roadmap to reflect the `0.36.0` wildcard-bag baseline.
